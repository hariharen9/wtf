Under the hood

Built for insane speed.

Every architectural decision prioritises throughput and portability — without a single CGO dependency.

Parallel Walker

Multi-worker goroutine pool saturates every CPU core simultaneously, walking the filesystem faster than any single-threaded traversal.

CPU → 100% parallel utilization

RAM Chunk Search

The full index is loaded into RAM once, then divided into CPU-aligned byte slices. Each core scans its own chunk with zero heap allocation.

index.db → RAM → N parallel slices

Zero CGO · Single Binary

100% pure Go. No C compiler required. One binary that runs anywhere.

Smart-Case & Fuzzy

Lowercase → case-insensitive. Mixed-case → exact match. Toggle fuzzy for boundary-bonus scoring.

Fuzzy mode

Atomic Index Writes

Writes to a temp file, then renames atomically. Searches are never interrupted mid-update — even when Ctrl+U rebuilds the entire index live.

Non-Blocking TUI

Bubble Tea MVU + Lipgloss styling. Index rebuilds are async tea.Cmd goroutines — the UI stays perfectly fluid while thousands of files are being walked.

Pipeline-Safe Output

Detects ModeCharDevice to strip all ANSI escape codes when piped, so wtf search app.js | grep src always outputs clean, raw paths.

How it works

The execution pipeline.

Two independent lifecycle paths: Write (indexing) and Read (searching), each optimised end-to-end for maximum throughput.

Allocation-Free Scanning

Byte-scanning never allocates on the heap. Only matching segments are converted to strings — GC pause time is effectively zero.

40% Smaller Binary

-ldflags "-s -w" strips the symbol table and DWARF debug info from the compiled binary, shrinking the download by over 40%.

CPU-Aligned Workers

Threading matches physical core count — maximises cache locality and keeps processors at 100% utilization during index scans without system thrashing.

Avoiding mmap

Go's native os.ReadFile reads a 50 MB index into RAM in 2–3ms. Parallel search on the raw byte slice is simpler and more portable than mmap across platforms.
